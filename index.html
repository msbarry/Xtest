What is Xtest?

Xtest is a scripting language tailored to testing Java code. It ships as an Eclipse plugin that lets you write .xtest files anywhere in your workspace in which test failures are highlighted just like compile errors.

Features

Simple, flexible syntax

Xtest has a simple syntax that eliminates Java boilerplate and lets you start writing tests immediately. The syntax is very similar to Java, but with additional expressions that make unit testing concepts part of the language.

IDE integration

Xtest is implemented using Xtext and thus enjoys full Eclipse IDE integration including syntax and error highlighting, hyperlinking, auto-complete, etc. so that using it feels no different than Java.

Automatic test execution

Xtest files sit in your Eclipse workspace in a passing or failing state and are run automatically by Eclipse any time a dependency changes to immediately notify you when a test has been broken.

Xbase goodies

Xtest is based on Xbase which makes it feel just like Java, with a few nice additions including extended type inference, custom overloaded operators, closures, enhanced switch expressions, and sugared getter/setter syntax to name a few.

JUnit integration

Xtest scripts can invoke and be invoked by JUnit tests. Xtest can be used to alert you when your existing JUnit test are failing or can be linked into your existing Junit test suite.

Private member access

Xtest allows you to access and modify private members of Java classes. Think of a tool for testing a circuit board that can probe voltages at internal nodes and insert erroneous states to see how the circuitry will respond - that is what Xtest allows you to do to Java code without changing your API.
